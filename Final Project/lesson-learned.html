Lessons I learned so far

This Semester was a journey to remember.

  • After completing this project, I tested my final project in different browsers, for example, Google Chrome, Mozilla Firefox, Microsoft Edge, and even on my cell phone. During the testing, I saw that my website fit and worked well on a small screen also because I used media queries; however, I noticed that the spacing did not look the same in all browsers. I figured out that testing on multiple browsers helped me become familiar with the importance of responsive design. I also noticed that responsive design makes websites adjustable in every devices and browsers.

  • In week 1:

    The first week was really challenging for me because it was my first experience learning such things. When I started learning, I learned simply how websites work and how we can manage files correctly. Similarly, I found the difference between local files and remote files, particularly on the web. I learned about URLs, and I knew the importance of web standards to display content in a user-friendly sequence. I practiced the proper way of managing files in VS Code. I also learned how to use AI to generate HTML code. I also learned about file extensions and the way of publishing a website in a user-friendly way and accessible to all users.

  • In week 2:

    On the second week, I felt a little bit relaxed before starting this HTML basic module; however, when I started, realized that this module has a lot of things to learn. In this module, I learned how developers create web pages and what HTML elements help to structure and provide content for web pages. I was familiar with different types of HTML tags, attributes, and syntax, and I also knew the importance of opening HTML files in different browsers. To make error-free code, I learned how to use an online validator, and the most important thing I learned is creating both external and internal links. I also figured out how HTML works with JavaScript, CSS, and XML.

  • In week 3:

    Week three was important for me because I learned the basics of CSS, which help to design HTML pages with colors, layout, fonts, spacing, etc. I also figured out CSS properties and the way of changing colors, text, and layout in webpages. During that time, I understood creating valid CSS stylesheets and the use of external stylesheets that give an attractive style to HTML pages. I also learned the use of element, class, ID, and descendant selectors. Overall, I learned the importance and use of CSS in web pages.

  • In week 4:

    In this week, I learned the use of visual and media elements in HTML pages. I learned about the different types of image formats and learned which one is right for an HTML page. I basically understood how to style pictures with CSS, using pictures as backgrounds of pages, and the important thing is how to make images work with links. I also understood the concept of applying visual effects using CSS3. Overall, I understood the importance of applying visuals correctly on the webpage that helps to use webpages in an accessible way.

  • In week 5:

    During this week, I understood the control of webpage layout by using CSS. Then I became familiar with the CSS box model, floating, margins, and many types of positioning such as fixed, relative, etc. I also learned how to make multiple columns, style navigation menus, and change how links look. In this module content, I simply learned responsive page design, along with media queries, proper images, the viewport meta tag, and Flexbox, which makes webpages run well.

  • In week 6:

    In week six, I learned how to make a website easy to use and accessible, especially for people who are differently able. I learned the difference between UI and UX, which work together on a webpage. UI refers to the look of a website, and UX refers to how it works and feels. I also learned about website organization, principles of visual design, and how to create simple and effective navigation on web pages. This module taught me how to create user -friendly webpages with features that make every reader comfortable using them.

  • In Week 7:

    In this module, I learned how to create HTML tables that help keep data organized. I gained knowledge about choosing the best HTML tags and CSS styles, which give webpages an attractive look. I also learned how to make webpages accessible for all users by improving table and form structure. Lastly, I learned how to choose appropriate CSS styles to add different colors, fonts, borders, spacing, and layout, making tables clean and neat.

  • In Week 8:

    During this module, I learned how JavaScript works on websites when users click, type something, and see changes quickly. It means that when users interact with a webpage, JavaScript can show popup information quickly and change text styles. I also learned creating basic JavaScript code using the script tag and the alert() method. Overall, this module taught me the importance of JavaScript on every webpages and helped me understand its elements, properties, methods, and functions clearly across different types of websites.

  • In Week 9:

    In this module, I learned about XML, which is used to store data sequentially and in a structured way. I also learned about SVG and web accessibility standards. I learned how to create and run SVG code as scalable graphics on web pages, and I also learned accessibility guidelines.

  • In Week 10:

    It was a last module of this course, so I learned a lot of thing that help me to be a developer this module, I learned about modern webpages development method, current but best using content management system, and I figure out the difference of client-side and web server-side programming.
